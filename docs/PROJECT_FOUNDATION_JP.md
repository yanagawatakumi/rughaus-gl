# RUGHAUS JP Horizon - 開発基盤ドキュメント（要件確定 v0.7）

最終更新日: 2026-04-28
対象テーマ: Shopify（標準状態 / 未カスタマイズ）

## 1. プロジェクト概要
- プロジェクト名: RUGHAUS GL
- 目的: 既存の国内向けサイトをベースに、海外向け販売サイトを構築する
- 実施方針: 国内ストア内の別テーマでPoC実装し、機能検証後に別Shopifyへ切り出して独立運用する

## 2. 開発フェーズ
- フェーズ1（PoC）: 国内ストア内の別テーマで必要機能の実現可否を検証
- フェーズ2（本構築）: PoC成果を別Shopifyストアに移行し、海外向けサイトとして独立運用
- 直近目標: フェーズ1で実課金まで見据えた技術検証を完了する
- 現行ストアプラン: Shopify Advanced（Plusではない）

## 2.1 開発環境前提（2026-04-13）
- Codex 開発環境に Shopify AI Toolkit（MCP + skills）を導入済みであること
- Shopify CLI は `3.93.2` 以上を前提とする（`shopify store auth/execute` を利用）
- ドメイン運用を分離すること
  - テーマ表示/プレビュー: `rughaus-gl.myshopify.com`
  - `shopify store` 実行: `xfxfwd-8p.myshopify.com`（恒久ドメイン）
- 日次運用手順の正本は `CODEX_SHOPIFY_AI_TOOLKIT_RUNBOOK.md` とする

## 3. PoC対象市場（確定）
- 重点対象国: アメリカ、香港、イタリア
- 広告検証対象: 上記以外の国からの流入も受け入れ、比較計測する
- 想定通貨: USD（基準）、EUR、KRW、TWD、HKD
- 通貨方針: 商品価格の基準はUSDで管理し、表示/決済は選択国に応じた通貨へ変換する
- 為替方針: 自動連動
- 通貨運用（ローンチ時）: ブラウザ/ロケーションを元に通貨を自動判定し、ユーザーがヘッダー等から手動変更できる
- 言語運用（ローンチ時）: ブラウザ言語/ロケーションを元に表示言語を自動判定し、ユーザーが手動変更できる
- ローンチ対応言語: English, Deutsch, Francais, Italiano, Espanol, 繁體中文, 한국어
- 翻訳方針: ローンチ対応言語の対象範囲で翻訳漏れゼロを必須化する

## 4. 商品スコープ（確定）
- メイン商材: ラグ
- 商品タイプ:
  - 既製品ラグ
  - サイズオーダーラグ（特注）
- サイズオーダーラグ:
  - 素材・カラーはバリエーションで管理
  - 価格は面積単価方式（単位価格 x 面積）
  - 形状/比率は両対応（固定比率 + 片側入力、自由入力）
  - PoC想定バリエーション: 1:1、2:3、自由

## 4.1 サイト構造（IA）
- ページ構成とヘッダー構成は `SITE_STRUCTURE_IA.md` を正本として管理する
- ローンチ時から言語/通貨セレクタを有効化し、ナビゲーションは全言語で同等導線を維持する
- IAの更新時は、ヘッダー導線とテンプレート構成を同時に見直す

## 5. 機能要件（PoC優先）
- FR-01: 国ごとに表示通貨を制御できること
  - 自動判定された通貨が初期表示され、ヘッダー等の手動選択で上書きできること
  - 切替後通貨が一覧/詳細/カート/チェックアウトに一貫反映されること
- FR-02: ローンチ時から対応言語で表示できること
  - 自動判定された言語が初期表示され、手動選択で上書きできること
  - 対応言語の対象範囲で翻訳漏れがないこと
- FR-03: 国ごとに商品/コレクションの表示・非表示を制御できること
  - 理想: 商品単位とコレクション単位の両方
  - 最低ライン: どちらか一方で運用可能であること
  - 非公開対象へのアクセス挙動はShopify Markets/Catalog標準挙動に準拠する
- FR-04: サイズオーダーラグの商品ページで、メタオブジェクトを利用した注文UIを実装できること
  - 素材/カラー: バリエーション
  - 比率選択: 1:1、2:3、自由（PoC）
  - PoC要件: 見積表示に留めず、実課金額へ正しく反映できること
  - 実装方針: 既存のサイズオーダーアプリを利用する
- FR-05: コレクションページでカラースウォッチを表示し、選択カラーに応じて商品画像が切り替わること
  - 実装方針: バリアント画像連動
- FR-06: 全商品ページで納期表示、注意文、返品条件を必須表示できること
- FR-08: 法人向け価格として「通常価格」と「20%OFF価格」をサイト内で並行表示し、決済は20%OFF価格で成立すること
  - サイズオーダー商品でも同一ルールで20%OFFが適用されること
  - 実装方針: 既存のB2B/法人価格アプリを利用する
- FR-08a: 価格の正本は `通常価格 + LPOサイズオーダー追加料金` を基準とし、最後にB2B割引を適用した金額とする
  - LPO追加料金は隠し商品行としてカートへ反映されること
  - B2B割引はドラフト経由のチェックアウトで適用されること
- FR-08b: 法人割引の初期条件は顧客タグで判定し、国条件は問わない
- FR-09: IA定義に沿ってページ構成とヘッダーナビゲーションを実装し、主要導線を最短化すること
- FR-10: Custom商品の運用ルールを明示し、購入前に必須表示すること
  - 返品不可
  - キャンセル不可
  - 納期遅延時は他商品と同様対応

## 6. 運用方針（確定）
- 開発基盤: Codex + Shopify AI Toolkit（MCP + skills）を標準環境として運用する
- 国別表示制御: Shopify Markets + カタログ運用を第一候補
- 国別表示制御の優先順位: まずコレクション単位を成立させる（商品単位は次点）
- 翻訳運用: テーマ内固定文言、動的文言、アプリ表示文言を含め翻訳漏れゼロを必須化
- 翻訳アプリ方針: 翻訳カバレッジが高く、未翻訳残りを管理できるアプリのみ採用する
- 翻訳アプリ構成（確定）:
  - Shopify標準翻訳アプリ（Translate & Adapt）は無料枠の制約を前提に利用する
  - 多言語運用（ローンチ対応7言語）を成立させるため、`T Lab - AI Language Translate` を併用する
  - 自動翻訳は下書き生成として利用し、公開前に未翻訳ゼロの監査を必須化する
- 標準機能優先: Shopify標準で不足する場合のみアプリ導入
- アプリ連携方針: サイズオーダーアプリとB2Bアプリの併用時に、表示価格と課金価格の一致を最優先で検証する
- 暫定採用アプリ:
  - Matrixify（データ入出力・移行補助）
  - B2B BSS（法人向け価格・卸売運用）
  - Live Product Options（サイズオーダー/カスタムオーダー）
  - T Lab - AI Language Translate（多言語翻訳運用）

## 8. 非機能要件
- 速度: 主要ページで体感遅延を抑える（画像最適化、JS肥大化抑制）
- 保守性: セクション/スニペット単位で責務分離
- 可読性: 変更理由をコードコメントとドキュメントに記録
- 安全性: 本番反映前のステージング確認を必須化
- 編集運用性: 非エンジニア運用を前提に、管理画面（Theme Editor）から更新が必要な領域はセクション/ブロック/設定で編集可能に実装する

## 9. 実装仕様（確定）
- D-01: 言語構成
  - ローンチ対応: EN / DE / FR / IT / ES / ZH-TW / KO
  - 追加言語はローンチ後に需要ベースで順次拡張
  - 翻訳運用は `Translate & Adapt + T Lab - AI Language Translate` の併用を標準とする
- D-02: 初期選択ロジック
  - ブラウザ言語/ロケーションを元に言語・通貨を自動判定する
  - 自動判定は完全一致を要件とせず、ユーザーがヘッダー等で手動上書きできることを優先する
  - 手動変更した言語・通貨は主要導線で即時反映される
- D-03: 国別表示制御の優先順位
  - 先にコレクション単位を実装し、余力があれば商品単位まで拡張する
- D-04: サイズオーダー入力方式
  - 固定比率型（縦入力 -> 横自動算出）と自由入力型（縦横入力）の両対応
  - 固定比率はPoCで1:1、2:3を実装
  - 入力方式はPoCで固定し、途中変更しない
- D-05: 面積単価計算ルール
  - 内部計算基準は m2
  - 単位表示は国または手動選択に応じて cm/inch を切り替える
  - 端数は四捨五入
  - 入力刻みは 5cm または 2inch
  - 最小寸法/最大寸法はメタオブジェクトで管理可能にする
- D-06: 必須表示文言の管理方法
  - 納期/注意文/返品条件は商品メタフィールド管理
  - 共通表示が必要な文言は共通オブジェクトを作成し、商品メタフィールドから参照する
- D-07: PoC対象商品
  - サイズオーダー商品を対象に検証する
  - 想定仕様は「比率 1:1 / 2:3 / 自由 + サイズオーダー」
- D-08: 非公開対象の直リンク時挙動
  - テーマ側で独自ガードは持たず、Markets/Catalog標準挙動に委譲する
- D-09: 翻訳完了の品質基準
  - 一覧/詳細/カート/チェックアウト導線/エラー表示/通知表示を翻訳対象とする
  - ローンチ対応言語で未翻訳ゼロを合格条件とする
- D-10: 法人20%OFFの価格表示仕様
  - 商品ページと一覧ページで「通常価格」と「法人価格（20%OFF）」を表示する
  - カート/チェックアウトでは法人価格で課金される
  - 二重割引を防ぐため、20%OFFの適用ポイントは1系統に固定する
- D-11: サイズオーダー価格と法人割引の計算順序
  - 1) 通常商品価格にLPOサイズオーダー追加料金（隠し商品行）を加算
  - 2) 加算後の合計にB2B割引（ドラフト経由）を適用
  - 3) 端数処理を適用し、表示額と課金額を一致させる
- D-12: プラン・機能前提
  - PoCはShopify Advanced前提で実施する
  - サイズオーダーは既存アプリ、法人価格は既存B2Bアプリで実装する
  - Shopify Plus限定機能を必須前提にしない
- D-13: アプリ責務分担（固定）
  - Live Product Options: 面積計算、入力バリデーション、サイズ由来の価格反映
  - B2B BSS: 法人20%OFFの表示・課金適用
  - テーマ: 2価格表示UI、文言表示、導線制御
- D-14: 価格適用順序（アプリ連携）
  - 1) Live Product Optionsでサイズオーダー追加料金を隠し商品行として生成
  - 2) B2B BSSで顧客タグを判定し、ドラフト経由で割引適用
  - 3) カート/チェックアウトで最終課金額を確定
- D-15: データ運用アプリ
  - Matrixifyを利用し、商品/メタフィールド/コレクションの入出力と移行作業を管理する
- D-16: 言語追加トリガー
  - 受注数、CVR、問い合わせ言語要望を指標とし、ローンチ対応外の追加言語を判断する
- D-17: 割引適用条件（初期）
  - 顧客タグで判定し、国条件は問わない
  - 将来は「顧客タグ + 国」の複合条件へ拡張可能にする
- D-18: 注文データ保存
  - サイズオーダー関連データはLive Product Options標準生成データを利用する
  - 追加の独自保存仕様は初期スコープ外とする
- D-19: Custom商品運用ルール
  - 返品不可
  - キャンセル不可
  - 納期遅延時は他商品と同様の対応
- D-20: マーケット運用（一般初期値）
  - Markets標準設定を利用して販売対象国・通貨・税・配送を管理
  - 重点対象国（US/HK/IT）を優先しつつ、広告流入国は同条件で受け入れる
  - 非公開商品/コレクションへのアクセスはMarkets/Catalog標準挙動で統一する
- D-21: 計測設計（一般初期値）
  - GA4を基準に計測し、広告媒体はUTMで流入を識別する
  - 最低限イベント: page_view, collection_view, product_view, size_input, add_to_cart, begin_checkout, purchase, trade_form_submit
- D-22: Topページの実装方針（Theme Editor優先）
  - Topページの主要領域は、ハードコード優先ではなくセクション/ブロックで編集可能に設計する
  - 特に運用変更が多い領域（例: TRENDING RUGS、カテゴリ導線、サポート導線）はブロック駆動を必須とする
  - 開発者のみが編集可能な実装（固定Liquid配列・固定カード）は、運用要件がない場合の最小範囲に限定する
  - `home-nk` のような複合セクションは分割し、`Hero / Categories / Trending / Story / Support` を独立セクションとして管理する
  - Topテンプレートは左カラムのセクション並び替えでレイアウト順を変更できる構成を標準とする
- D-23: 全ページ共通の実装方針（Theme Editor優先）
  - Topページに限らず、About / Trade / Support / Collection / Product を含む全ページで、基本はセクション/ブロック分割で構築する
  - 1ページ1セクションのモノリシック実装は原則禁止とし、例外は仕様上分割不能な最小範囲のみ許可する
  - テンプレート（`templates/*.json`）は、Theme Editor左カラムでセクションの追加・削除・並び替えが可能な構成を標準とする
  - 将来運用で更新が想定される文言・画像・リンクは、settingsまたはblocksへ必ず露出する
- D-24: カスタムラグ単価データモデル（段階導入）
  - 方針: 「まずAで開始し、将来Bへ移行」する
  - A（Phase 1）:
    - 商品メタフィールド `custom.price_per_m2`（`number_decimal`）を正本として利用する
    - 納期は既存の `custom.pdp_delivery_shipping` を正本として継続利用する
    - 表示はテーマ側で市場に応じて単位のみローカライズする（段階導入は `NK Centered Gallery` から開始）
      - 内部基準単位は m2
      - 価格表記は全市場で USD 固定とする
      - US表示時は `ft2` のみ表示し、`USD/m2` から換算（`1 m2 = 10.7639 ft2`）
      - 非US表示時は `USD/m2` 表示とする
    - 丸めは小数1桁（四捨五入）とする
    - 為替換算は行わない（USD固定）
  - B（将来移行）:
    - 商品はティア選択（例: A/B/C）のみを保持し、価格表は metaobject へ分離する
    - 商品メタフィールド `custom.price_tier` と価格表metaobject参照へ置換する
  - 実装制約:
    - 画面側は価格取得をスニペット経由に統一し、A→Bの切替をデータ源差し替えのみで完了できる構造にする
    - 表示価格ロジックと、LPO/B2Bによる実課金ロジックの整合確認をリリースゲートに含める
